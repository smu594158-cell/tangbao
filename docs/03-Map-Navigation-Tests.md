# 地图与路线模块测试用例

## 1. 单元测试用例设计 (React Testing Library + Jest)

### 用例 1.1：正常导航场景
- **预置条件**: 用户处于登录状态，进入“地图与路线”页面。
- **操作步骤**: 
  1. 在起点输入框输入“西湖”。
  2. 在终点输入框输入“灵隐寺”。
  3. 选择“步行”作为出行方式。
  4. 点击“开始规划”按钮。
- **期望结果**:
  - `axios.get('/api/v1/geo/poi/search')` 被成功调用两次，返回起点和终点的坐标。
  - `axios.get('/api/v1/geo/route/plan')` 被成功调用，返回规划详情（总距离、耗时、步骤）。
  - 页面渲染出路线结果卡片，显示具体数值。
  - 触发 `AMap.Walking` 的 `search` 方法，地图组件加载高德路线图层。

### 用例 1.2：网络异常场景
- **预置条件**: 模拟断网或服务器返回 500 状态码。
- **操作步骤**: 
  1. 输入起点与终点，点击“开始规划”。
- **期望结果**:
  - axios 请求被 reject。
  - 页面右上角弹出全局提示 `message.error('网络请求失败')`。
  - 规划结果卡片不显示。
  - 按钮的 Loading 状态在请求结束后自动恢复。

### 用例 1.3：权限拒绝/未知地点场景
- **预置条件**: 输入一个系统无法通过 POI 解析获取经纬度的模糊地点。
- **操作步骤**: 
  1. 起点输入“不存在的虚拟地点”。
  2. 点击“开始规划”。
- **期望结果**:
  - 第一次 POI 查询接口返回空数组 `[]`。
  - 页面弹出提示 `message.error('无法找到起点位置')`。
  - 流程提前中断，不会继续调用路径规划接口和地图渲染 API。

## 2. 集成/端到端测试用例 (E2E)

### 用例 2.1：高德地图 App 唤起与 H5 降级测试
- **场景**: 测试移动端设备的混合导航唤起逻辑。
- **测试设备**: 
  1. 安装了高德地图 App 的手机浏览器。
  2. 未安装高德地图 App 的手机浏览器或 PC 浏览器。
- **操作步骤**:
  1. 成功生成一条“驾车”路线。
  2. 点击页面下方的“一键导航 (支持唤起高德App)”按钮。
- **期望结果**:
  - **有 App 设备**: 浏览器触发 `amapuri://route/plan/?...` Scheme 协议，系统弹窗询问是否打开“高德地图”，确认后跳转至 App 且起点终点数据自动带入。
  - **无 App 设备**: 浏览器触发 Scheme 协议失败后，静默等待 2 秒钟（`setTimeout` 降级逻辑），随后打开新标签页跳转至 `https://uri.amap.com/navigation?...` 的高德 H5 导航页面。

### 用例 2.2：高德 JS API 插件异步加载测试
- **场景**: 测试前端地图插件按需加载的健壮性。
- **操作步骤**:
  1. 进入页面，地图基础瓦片加载完成。
  2. 多次切换“出行方式”（驾车 -> 公交 -> 骑行），分别点击规划。
- **期望结果**:
  - 高德 JS API 的 `AMap.plugin` 能够正确且独立地加载对应的插件（`AMap.Driving`, `AMap.Transfer`, `AMap.Bicycling`）。
  - 每次重新规划前，页面均调用了 `routePluginRef.current.clear()` 成功清除了上一条旧的路线，确保地图上不会出现多条路线重叠。
